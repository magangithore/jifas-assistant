## 7?? BEFORE vs AFTER COMPARISON

### BEFORE (web.config)
```xml
<configuration>
  <configSections>
    <section name="entityFramework" ... />
  </configSections>
  <appSettings>
    <add key="Gemini:ApiKey" value="AIzaSy..." />
    <add key="Gemini:Model" value="gemini-2.0-flash" />
    <add key="Chat:DefaultErrorMessage" value="..." />
  </appSettings>
  <entityFramework>
    ...
  </entityFramework>
</configuration>
```

### AFTER (appsettings.json + AppSettings.cs)
```json
{
  "Gemini": {
    "ApiKey": "${GEMINI__APIKEY}",
    "Model": "gemini-2.0-flash"
  },
  "Chat": {
    "DefaultErrorMessage": "..."
  }
}
```

```csharp
// Access dengan type-safe
var apiKey = _appSettings.Gemini.ApiKey;
var errorMsg = _appSettings.Chat.DefaultErrorMessage;
```

## 1??1?? TROUBLESHOOTING DECISION TREE

```
Configuration tidak ter-load?
?? Apakah file ada?
?  ?? Yes ? Next step
?  ?? No ? Create file
?
?? Apakah format JSON valid?
?  ?? Yes ? Next step
?  ?? No ? Fix JSON syntax
?
?? Apakah environment variable set?
?  ?? Yes ? Check naming (use __)
?  ?? No ? Set environment variable
?
?? Apakah registered di DI?
   ?? Yes ? Problem solved
   ?? No ? Add to Program.cs


Value always null?
?? Apakah key name benar?
?  ?? Case sensitive? Check!
?  ?? Hierarchy benar? (use : untuk nesting)
?
?? Apakah accessing dengan benar?
?  ?? IOptions<T>.Value.Property
?  ?? IConfiguration["Key:SubKey"]
?  ?? AppSettings.Section.Property
?
?? Apakah value overridden?
   ?? Check hierarchy: Env > Secrets > {Environment}.json > default
```
