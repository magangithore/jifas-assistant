Master Division

Deskripsi

Modul Master Division digunakan untuk mendata dan mengelola unit bisnis (divisi) yang berada di bawah masing-masing company dalam ekosistem JIFAS. Data ini memastikan struktur organisasi tersusun secara hierarkis sehingga laporan dan hak akses dapat ditentukan secara tepat.

Fungsi
Menyediakan database terpusat untuk seluruh data divisi di perusahaan.
Menjadi referensi utama untuk pengelolaan struktur organisasi, pelaporan, dan pemetaan data master lain (departement, employee, dsb).
Mendukung validasi dan konsistensi data divisi pada seluruh modul JIFAS.

Tujuan
Mencegah duplikasi data divisi di sistem.
Memudahkan proses pencarian, pelaporan, dan integrasi data antar modul.
Menjamin akurasi dan legalitas data divisi yang digunakan dalam transaksi dan pelaporan.

Berikut adalah rincian role yang memiliki otoritas untuk melakukan pembuatan atau perubahan data Division:
WMTR: Webmaster
FINA: Finance
SPVR: Supervisor Finance
HDFA: Head Finance
MGFA: Magang Finance

Create (Menambah Division Baru)
1. Klik tombol Create (ikon "+") di bagian kiri atas grid.
2. Jendela input akan muncul.
3. Lengkapi seluruh field yang diperlukan sesuai petunjuk.
4. Setelah data terisi benar, klik tombol Create untuk menyimpan.

Update (Memperbarui Data Division)
1. Cari division yang ingin diubah pada grid.
2. Pada kolom Action di sisi kanan, klik ikon Edit (pena).
3. Jendela edit akan muncul.
4. Perbarui field yang diperlukan.
5. Klik tombol Update untuk menyimpan perubahan.

Delete (Menghapus Division)
1. Cari division yang ingin dihapus pada grid.
2. Pada kolom Action di sisi kanan, klik ikon Delete (tempat sampah).
3. Akan muncul jendela konfirmasi penghapusan.
4. Jika yakin ingin menghapus, klik Yes untuk melanjutkan.

Berikut adalah rincian field yang tersedia untuk pengelolaan data Division berdasarkan informasi yang Anda berikan:
Division Code: Menggunakan Kolom Teks. Wajib diisi (Required) dan harus bersifat Unik. Digunakan untuk memasukkan kode unik divisi dengan batas maksimal 4 karakter.
Division Name: Menggunakan Kolom Teks. Wajib diisi (Required). Digunakan untuk memasukkan nama divisi sesuai dengan dokumen internal perusahaan.
Company: Menggunakan Dropdown Pilihan. Wajib diisi (Required). Digunakan untuk memilih perusahaan induk tempat divisi tersebut bernaung.
Is Active: Menggunakan Tombol On/Off (Switch). Berfungsi untuk menentukan status aktif divisi dengan nilai bawaan (Default) adalah On.

Kolom Teks Biasa: Tempat mengetik data secara bebas (kode, nama divisi, dll).
Dropdown Pilihan: Daftar perusahaan yang sudah tersedia di Master Company, tinggal klik pilih.
Tombol On/Off: Untuk mengaktifkan atau menonaktifkan divisi.

Fitur-fitur Tabel Data

Mengurutkan Data
Klik pada judul kolom untuk mengurutkan data dari A-Z atau Z-A.
Standar: data diurutkan berdasarkan nama divisi dari Z-A.

Mencari Data
Setiap kolom memiliki kolom pencarian di baris pertama tabel.
Ketik kata kunci untuk mencari division.
Contoh: ketik "MKT" untuk mencari semua division dengan kode "MKT".

Membaca Data Per Halaman
Data ditampilkan per halaman (standar: 10 data per halaman).
Gunakan tombol navigasi di bawah tabel untuk pindah halaman.
Bisa juga pilih jumlah data yang ingin ditampilkan per halaman.

Mengubah Lebar Kolom
Arahkan mouse ke garis pemisah kolom, lalu drag untuk mengubah lebar kolom.

Tombol Tambah Data
Tombol Create ada di atas tabel untuk menambah division baru.

Pesan dan Konfirmasi

Pesan Sukses: Setelah berhasil tambah/ubah/hapus, akan muncul notifikasi sukses.
Pesan Error: Jika ada masalah, akan tampil pesan error yang menjelaskan masalahnya beserta kode error dari server.
Konfirmasi Sebelum Hapus: Sebelum menghapus data, sistem akan menanyakan "Apakah Anda yakin ingin menghapus?" dengan menampilkan kode dan nama division yang akan dihapus untuk mencegah kesalahan.

Catatan Tambahan

Pastikan setiap divisi yang dibuat mereferensikan company yang aktif di Master Company.
Gunakan konsistensi penamaan supaya pencarian dan pelaporan antar divisi lebih mudah.
Division Code bersifat read-only (tidak bisa diubah) setelah data dibuat untuk menjaga integritas data.
Setiap perubahan division akan langsung tersimpan dan dapat digunakan untuk departement di bawahnya.
