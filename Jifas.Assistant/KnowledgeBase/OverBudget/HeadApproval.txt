KNOWLEDGE BASE OVER BUDGET APPROVAL MODULE

DESKRIPSI UMUM

Module Over Budget Approval adalah modul yang digunakan untuk mengelola persetujuan atau approval terhadap dokumen invoice dan PUM yang amount nya melebihi budget yang tersedia. Module ini memiliki dua tabrifikasi utama yaitu Invoice Over Budget untuk approval invoice yang over budget dan PUM Over Budget untuk approval PUM yang over budget. Module ini adalah tahap kritikal dalam workflow approval karena memastikan semua dokumen yang over budget mendapat approval eksplisit dari head of department sebelum diproses lebih lanjut. Module ini memungkinkan head untuk membuat keputusan strategic tentang budget exception dan prioritas spending.

SASARAN PENGGUNA

- Head of Department yang memiliki budget authority untuk approve over budget items
- Finance Head yang melakukan final approval over budget approvals
- Finance Manager yang melakukan escalation untuk over budget exceptions
- Accounting Manager yang track over budget approvals
- Financial Planner yang monitor budget utilization trends
- Internal Audit yang verify over budget approval decisions

FITUR UTAMA - INVOICE OVER BUDGET

1. INVOICE OVER BUDGET GRID

Feature ini menampilkan daftar semua invoice documents yang memiliki amount melebihi available budget untuk COA yang dipilih. Grid menampilkan informasi penting seperti tanggal dokumen, nomor invoice, kategori, deskripsi, COA code, amount, dan status approval.

- Filter otomatis untuk date, document number, category
- Sorting berdasarkan tanggal dokumen (DESC)
- Pagination dengan default 10 items per halaman
- Warna highlight untuk rows berdasarkan status (Need Approval = green)
- Export to Excel functionality untuk audit tracking
- View/Approval button untuk membuka detail form

2. INVOICE DETAIL APPROVAL FORM

Form detail yang terbuka untuk setiap invoice over budget yang dipilih. Form menampilkan informasi lengkap invoice termasuk header data dan supporting documents.

- Multiple tabs untuk Approval dan File information
- Header section dengan informasi dokumen (date, COA code, amount, remaining budget, PPN, PPH)
- Vendor, Applicant, Division, Approve By information
- Description field
- File tab menampilkan supporting documents list
- Approve dan Reject buttons

3. DOCUMENT VERIFICATION TAB

Feature dalam approval form yang menampilkan list dokumen pendukung yang sudah di-attach untuk invoice tersebut.

- Grid berisi daftar dokumen yang sudah uploaded
- Document information tracking
- File attachment references
- Read-only display untuk document review

4. APPROVAL DECISION WORKFLOW

Feature untuk melakukan approval atau rejection terhadap invoice over budget.

- Confirmation dialog untuk approve action
- Rejection reason dialog untuk reject action
- API integration untuk status update ke backend
- Audit trail untuk setiap decision

5. INLINE DETAIL GRID

Feature untuk menampilkan detail dokumen pendukung dalam grid format.

- Display dokumen type, code, nomor
- Information field untuk deskripsi
- File attachment references
- Read-only presentation untuk review

FITUR UTAMA - PUM OVER BUDGET

1. PUM OVER BUDGET GRID

Feature menampilkan daftar semua PUM documents yang memiliki amount over budget. Similar dengan invoice grid tetapi untuk PUM document type.

- Filter otomatis untuk date, document number, category (PP type)
- Sorting berdasarkan tanggal (DESC)
- Pagination dengan default 10 items per halaman
- Color highlighting untuk status
- Export to Excel functionality
- Approval button untuk membuka detail form

2. PUM DETAIL APPROVAL FORM

Form detail untuk PUM over budget approval dengan structure mirip invoice.

- Approval tab dengan header information
- File tab dengan document attachments
- Tab strip untuk organization
- Multiple information fields untuk PUM-specific data

3. DOCUMENT ATTACHMENT DISPLAY

Feature khusus untuk PUM yang menampilkan attached documents dengan visual preview.

- Grid untuk document attachments
- Image preview untuk PNG/JPG files
- Download button untuk document files
- File extension detection untuk proper display
- Inline template untuk document rendering

4. PUM APPROVAL DECISION

Feature untuk approve atau reject PUM over budget.

- Confirmation dialog untuk approval
- Rejection reason capture untuk rejection
- API call dengan bgAppr dan bgIsOk flags
- Status update tracking

5. FILE MANAGEMENT

Feature untuk manage file attachments dalam PUM approval.

- Display file paths
- Extension-based rendering (images vs files)
- Clickable links untuk file access
- File type identification

WORKFLOW INVOICE OVER BUDGET

Berikut adalah flow lengkap proses Invoice Over Budget Approval:

TAHAP 1 PREPARATION

- User membuka Invoice Over Budget tab dari module
- System menampilkan grid dengan daftar invoice yang memiliki over budget amount
- Grid di-populate dari endpoint API Invoice Over Index
- Hanya invoice dengan amount lebih besar dari remaining budget ditampilkan
- List di-sort berdasarkan tanggal dokumen (terbaru di atas)
- Pagination controls ditampilkan

TAHAP 2 FILTERING DAN SEARCHING

- User dapat filter berdasarkan date range
- User dapat search berdasarkan document number
- User dapat filter berdasarkan category
- Filter dilakukan row-based
- Grid auto-refresh dengan filter criteria

TAHAP 3 INVOICE SELECTION

- User memilih salah satu invoice dari grid dengan mengklik row
- User mengklik Approval button untuk membuka approval form
- Popup window dengan title Invoice Approval Over Budget dibuka
- Form siap untuk review dan decision

TAHAP 4 FORM INITIALIZATION

- System melakukan fetch terhadap detail over budget invoice
- Data di-populate ke form fields dari API response
- Tab strip di-initialize dengan Approval dan File tabs
- Document grid di-initialize untuk supporting documents

TAHAP 5 DETAIL REVIEW - APPROVAL TAB

- Left section menampilkan:
  - Document Date (readonly)
  - COA Code (readonly)
  - Amount (readonly dengan currency format)
  - Remaining Budget (readonly - showing available budget)
  - PPN amount (readonly)
  - PPH amount (readonly)

- Right section menampilkan:
  - Vendor name
  - Applicant (staff name yang membuat invoice)
  - Division code
  - Approve By (head department name)
  - Description

TAHAP 6 DOCUMENT VERIFICATION - FILE TAB

- User dapat click File tab untuk review supporting documents
- Detail grid menampilkan:
  - Document Type (jenis dokumen requirement)
  - Code (singkat code)
  - Document Number
  - Information/Deskripsi
- Grid display untuk review documents attached

TAHAP 7 DECISION MAKING

User memilih salah satu dari dua path:

PATH A: APPROVE OVER BUDGET

- User mengklik Approve button
- Confirmation dialog terbuka dengan text
  "Anda yakin ingin menyetujui dokumen [DOCNUMBER] ini?"
- User click Done untuk confirm approval

TAHAP 8A APPROVAL PROCESSING

- API call dilakukan ke Invoice Over UpdateStatusPd
- Parameters dikirim:
  - action: A (untuk Approve)
  - recNo: record number dari invoice
  - docNo: invoice document number
  - rejectNote: empty string
- Authorization header di-include
- Progress indicator ditampilkan

TAHAP 8B APPROVAL COMPLETION

Success scenario:
- Success message ditampilkan
- Main grid di-refresh untuk remove approved invoice
- Invoice status updated ke approved
- Document forwarded ke next workflow stage

Error scenario:
- Error message ditampilkan dengan failure reason
- User dapat retry approval

PATH B: REJECT OVER BUDGET

- User mengklik Reject button
- Rejection reason prompt dialog terbuka dengan text
  "Kenapa Permintaan [DOCNUMBER] senilai [AMOUNT] tidak disetujui?"
- User harus input rejection reason

TAHAP 9B REJECTION REASON VALIDATION

- System validate rejection reason tidak kosong
- If empty: error message "Please fill the reason to reject this document!"
- If filled: proceed ke confirmation

TAHAP 10B REJECTION CONFIRMATION

- Alert dialog menampilkan "Reason to reject this document is: [REASON]"
- User acknowledge alert
- Confirmation dialog terbuka untuk final confirmation

TAHAP 11B REJECTION PROCESSING

- API call dilakukan ke Invoice Over UpdateStatusPd
- Parameters dikirim:
  - action: X (untuk Reject/Cancel)
  - recNo: record number
  - docNo: document number
  - rejectNote: rejection reason dari user
- Authorization header di-include
- Progress indicator ditampilkan

TAHAP 12B REJECTION COMPLETION

- Success message ditampilkan dengan status update
- Main grid di-refresh
- Invoice status updated ke rejected
- Rejection reason logged dalam system untuk audit

TAHAP 13 WORKFLOW COMPLETION

- Detail form ditutup setelah approval atau rejection
- Main grid di-refresh untuk show updated status
- User dapat melanjutkan dengan invoice berikutnya

VALIDATION RULES

RULE 1 DOCUMENT SELECTION

- User harus select one document dari grid sebelum open approval form
- Only documents dengan over budget amount dapat di-select
- Document status harus memungkinkan untuk approval

RULE 2 APPROVAL CONFIRMATION

- Approval action memerlukan confirmation dialog
- User harus click Done dalam confirmation dialog
- Document number harus di-display dalam confirmation

RULE 3 REJECTION REASON

- Rejection action memerlukan reason input
- Rejection reason tidak boleh kosong atau blank
- User harus provide meaningful reason untuk audit trail

RULE 4 API PARAMETER VALIDATION

- Invoice over: require action, recNo, docNo, rejectNote parameters
- PUM over: require docNo, coa_code, bgAppr, bgIsOk, apprNote parameters
- Authorization token harus valid dalam header

RULE 5 STATUS UPDATE

- After successful approval: status updated ke approved
- After successful rejection: status updated ke rejected dengan reason logged
- Grid automatically refresh untuk reflect new status

RULE 6 DOCUMENT REQUIREMENT

- Each over budget document harus have COA code
- COA code mendetermine budget constraint
- Amount harus clearly exceed remaining budget untuk appear dalam grid

TROUBLESHOOTING GUIDE

ISSUE 1 GRID TIDAK LOADING

Symptom: Over budget grid menampilkan kosong atau loading forever
Cause: API endpoint tidak accessible atau data issue
Solution:
- Verify API URL correct untuk Invoice/Over/Index
- Check network connectivity
- Verify company code sudah set di system
- Refresh page dengan F5
- Check browser console untuk error messages

ISSUE 2 APPROVAL FORM TIDAK MEMBUKA

Symptom: Saat click Approval button form tidak appear
Cause: PopUp blocker atau JavaScript error
Solution:
- Disable popup blocker untuk website
- Clear browser cache
- Try dengan browser berbeda
- Check console untuk JavaScript errors
- Verify authorization token valid

ISSUE 3 CONFIRMATION DIALOG TIDAK MUNCUL

Symptom: Saat click Approve button dialog tidak appear
Cause: Kendo dialog tidak initialized
Solution:
- Refresh page
- Check Kendo library sudah loaded
- Try browser berbeda
- Verify JavaScript enabled di browser

ISSUE 4 REJECTION REASON TIDAK ACCEPTED

Symptom: Reject action gagal atau reason tidak ter-save
Cause: Validation error atau API issue
Solution:
- Verify rejection reason tidak kosong
- Check error message di alert
- Ensure reason valid string format
- Verify API endpoint accessible
- Check response dari backend

ISSUE 5 GRID TIDAK REFRESH SETELAH ACTION

Symptom: After approval atau rejection, grid tidak update
Cause: Refresh function issue atau backend delay
Solution:
- Wait untuk backend processing complete
- Manually refresh dengan F5
- Check browser console untuk errors
- Verify API success response
- Retry manual refresh

ISSUE 6 DOCUMENT GRID TIDAK MENAMPILKAN FILES

Symptom: File tab kosong atau documents tidak showing
Cause: Data fetch gagal atau template issue
Solution:
- Refresh form
- Close dan open form kembali
- Check API endpoint untuk document fetch
- Verify document attachments exist di backend
- Contact admin untuk data verification

ISSUE 7 IMAGE PREVIEW TIDAK RENDER

Symptom: Image files dalam File grid tidak menampilkan preview
Cause: File path issue atau rendering error
Solution:
- Check file path URL format correct
- Verify image file exist di server
- Check file permissions
- Try direct URL di browser untuk test
- Contact admin untuk path verification

ISSUE 8 APPROVAL ACTION GAGAL

Symptom: Approval atau rejection action failed dengan error
Cause: API error, validation failure, atau backend issue
Solution:
- Read error message carefully
- Verify all required fields filled
- Check authorization valid
- Verify document data intact
- Retry approval action
- Contact support dengan error details

ISSUE 9 FILTER TIDAK WORKING

Symptom: Filter criteria tidak bekerja untuk reduce data
Cause: Filter configuration issue
Solution:
- Clear filter dan reapply
- Refresh page
- Check filter value format correct
- Try berbeda filter criteria
- Verify data exist untuk selected criteria

ISSUE 10 EXPORT EXCEL INCOMPLETE

Symptom: Export result tidak include semua data atau cut off
Cause: Large dataset atau export limit
Solution:
- Apply filter untuk reduce data sebelum export
- Export dalam smaller batch dengan date range
- Contact admin untuk large data export
- Verify selection sebelum export
