## 1?? SQL SERVER (LocalDB) - For Conversation Storage

### Purpose
Menyimpan data percakapan, feedback, dan metrics yang terstruktur.

### Configuration
**File**: `appsettings.json`
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=(localdb)\\MSSQLLocalDB;Initial Catalog=JifasAssistant;Integrated Security=true;"
  }
}
```

### Database Entities (4 tables)
1. **Chats** - Conversation history
   - Id, UserId, Message, Response, CreatedAt
   
2. **KnowledgeBaseDocuments** - KB articles with embeddings
   - Id, Title, Content, Category, Embedding vector info
   
3. **UserFeedbacks** - User ratings & feedback
   - Id, ChatId, Rating, Comment, CreatedAt
   
4. **Metrics** - Performance & usage metrics
   - Id, MetricType, Value, Category, CreatedAt

### Database Creation
```bash
# Create migration
dotnet ef migrations add InitialCreate

# Apply migration
dotnet ef database update
```

**After this, `JifasAssistant` database will be created in LocalDB.**

## 2?? QDRANT (Docker) - For Vector Search

### Purpose
Menyimpan embeddings (vectors) dari knowledge base untuk semantic search.

### Configuration
**File**: `appsettings.json`
```json
{
  "Qdrant": {
    "Enabled": true,
    "Url": "http://localhost:6333",
    "CollectionName": "jifas_kb",
    "ApiKey": "your-secure-api-key-here",
    "EmbeddingDimensions": 384
  }
}
```

### How It Works
1. Knowledge Base documents ? Converted to embeddings (vectors)
2. Embeddings stored in Qdrant collection
3. User query ? Converted to embedding
4. Search in Qdrant for similar vectors
5. Return top K most relevant documents

### Docker Setup
**File**: `docker-compose.yml`
```yaml
qdrant:
  image: qdrant/qdrant:latest
  ports:
    - "6333:6333"
  volumes:
    - qdrant_data:/qdrant/storage
  environment:
    - QDRANT_API_KEY=your-api-key
```

**Start Qdrant:**
```bash
docker-compose up qdrant -d
```

**Verify:**
```bash
curl http://localhost:6333/health
# Response: {"ok":true}
```

## ?? KnowledgeBase Setup

### How Knowledge Base Documents Are Stored

#### 1. Document Structure (SQL Server)
```csharp
public class KnowledgeBaseDocument
{
    public int Id { get; set; }
    public string Title { get; set; }           // "Login JIFAS - Langkah-langkah"
    public string Content { get; set; }         // Full content
    public string Category { get; set; }        // "Getting Started"
    public string Source { get; set; }          // "FAQ", "Documentation"
    public DateTime CreatedAt { get; set; }
    public DateTime UpdatedAt { get; set; }
}
```

#### 2. Embedding Vector (Qdrant)
When a document is added:
```
Document: "Bagaimana cara login JIFAS?"
         ?
         ??? Convert to Embedding
         ?   (using Gemini API)
         ?
         ??? Create Vector: [0.123, -0.456, 0.789, ...]
         ?
         ??? Store in Qdrant with metadata:
             {
               "vector": [...],
               "payload": {
                 "doc_id": 123,
                 "title": "Login JIFAS",
                 "content": "...",
                 "category": "Getting Started"
               }
             }
```

#### 3. Search Process
```
User Query: "Bagaimana login JIFAS?"
         ?
         ??? Convert to Embedding
         ?   [0.125, -0.450, 0.792, ...]
         ?
         ??? Search in Qdrant
         ?   (Find similar vectors)
         ?
         ??? Return top 5 matches
             with similarity scores
```

### Seeding Knowledge Base

**File**: `run-migrations.sh` or `run-migrations.bat`
```bash
# This will be created for seeding KB
dotnet run -- seed-kb
```

## ?? Docker Configuration

### Services Available

```yaml

services:
  # Your API
  jifas-api:
    build: .
    ports:
      - "5000:80"
    depends_on:
      - sqlserver
      - qdrant
    environment:
      - ConnectionStrings__DefaultConnection=Server=sqlserver;...

  # SQL Server for data
  sqlserver:
    image: mcr.microsoft.com/mssql/server
    ports:
      - "1433:1433"
    environment:
      - ACCEPT_EULA=Y
      - SA_PASSWORD=YourPassword123!

  qdrant:
    image: qdrant/qdrant
    ports:
      - "6333:6333"
    volumes:
      - qdrant_data:/qdrant/storage
```

**Start everything:**
```bash
docker-compose up -d
```

**Services available at:**
- API: http://localhost:5000
- Qdrant: http://localhost:6333
- SQL Server: localhost:1433

## ?? Troubleshooting

### LocalDB Connection Error
```bash
# Check LocalDB is running
sqllocaldb info
sqllocaldb start MSSQLLocalDB

# Or reinstall
sqllocaldb delete MSSQLLocalDB
sqllocaldb create MSSQLLocalDB
```

### Qdrant Not Responding
```bash
# Check Docker is running
docker ps

# Restart Qdrant
docker-compose restart qdrant

# Check logs
docker-compose logs qdrant
```

### Migration Failed
```bash
dotnet ef migrations remove
dotnet ef migrations add InitialCreate
dotnet ef database update
```
